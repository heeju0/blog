EDA

💡 『KDT 프로그래머스 인공지능 데브코스』에서 공부중..👀

note

Titanic Dataset을 이용한다.

1. EDA#

시각화, 통계적 분석 등을 통해 데이터 자체만으로 인사이트를 얻어내는 접근법

EDA의 Process#

  1. 분석의 목적과 변수 확인
    분석의 목적을 명확히 하고, 각 컬럼의 데이터 타입, 의미 등을 파악한다.

  2. 데이터를 전체적으로 살펴보기
    데이터 간의 상관관계, 결측치, 데이터의 크기 등을 파악한다.

  3. 데이터의 개별 속성 파악하기
    각 데이터의 개별 속성과 각 속성이 적절히 매칭되어 있는지 파악한다.

2. EDA 실습#

라이브러리 준비#

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns
import warnings
warnings.filterwarnings('ignore')
%matplotlib inline
titanic_df = pd.read_csv('data/titanic_train.csv')

분석의 목적과 변수 확인#

분석의 목적#

탑승객 중에서 살아남은 사람들은 어떤 특징을 가지고 있었을지에 대해 분석하고자 한다.

변수 확인#

.describe()는 수치형 데이터에 대한 요약만 제공한다.

PassengerId: 인덱스와 같은 역할로, 생존 여부와는 관련이 없는 것 같아 보인다.
Survived: 평균이 0.5보다 작으므로 사망자가 생존자보다 많은 것이다.
Pclass: 범주형 데이터이기 때문에 요약은 의미가 없다.
Fare: mean과 max의 차이가 매우 크므로 편차가 매우 클 것으로 보인다.

+ Mission#

1. 본문에서 언급된 Feature를 제외하고 유의미한 Feature를 1개 이상 찾아봅시다.#

2. Kaggle에서 Dataset을 찾고, 이 Dataset에서 유의미한 Feature를 3개 이상 찾고 이를 시각화해봅시다.#

Hint#

  1. 데이터 톺아보기
    자료형, 결측치 확인한다.
    자료형을 바꿀 필요가 있는지 확인한다.
  2. 데이터에 대한 가설 세우기
    개인의 경험에 의해 도출되어도 상관 없다.
    명확할수록 좋다.
    ex) Survival 여부와 성별에는 상관관계가 있다.
  3. 가설을 검증하기 위한 증거 찾기
    .groupby()를 통해 그룹화된 정보에 통계량을 도입할 수 있다.
    .merge()를 통해 여러 개의 데이터프레임을 합칠 수 있다.
    시각화를 통해 보여줄 수 있다.
