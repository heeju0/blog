1. Git#

분산 버전 관리 시스템

중앙에 공용 코드 공간을 마련하고 각자 코드를 가져와서 작업한 뒤 비동기적으로 업로드할 수 있도록 함으로써, 독립적인 작업이 가능하고 저장소가 갱신되었을 때 쉽게 내 코드에 반영할 수 있다는 장점이 있다.

저장소 단위로 관리되며, 중앙에 원격 저장소를 두고 이를 바탕으로 여러 개의 로컬 저장소를 둔다. 이때, 로컬에서 작동하는 repository를 Git으로 관리하고, 원격으로 작동되는 repository를 Github가 관리하게 된다.

*repository: 저장소

Git 시작하기#

  1. Git 설치

  2. 로컬 저장소 생성

    $ git init
    $ ls -al # git은 숨김 폴더로 관리된다.

  3. git의 로컬 저장소에 아이디와 이메일 설정

    git config --global user.name "[아이디]"
    git config --global user.email "[이메일]"

Git 저장소에서 파일의 상태#


*REMOTE는 Github 부분

git은 기본적으로 commit이라는 단위로 진행된다.
모든 파일을 저장, 삭제, 생성하는 모든 기록하는 과정은 비효율적이기 때문에 git에서는 사용자가 임의로 저장하고자 하는 순간을 지정할 수 있으며, 이 단위를 commit이라고 한다.

처음에 commit에 반영하지 않은 파일(Unstaged 상태)들은 Working Directory에 존재하며, commit에 추가하고자 하는 파일들은 git add 명령어를 통해 Staged 상태가 된다.(Staging Area) git commit 명령어를 통해 해당 파일들은 Commited 상태가 되며(Repository), git은 commit된 파일 단위로 작동한다.

*add: commit하고자 하는 파일들을 지정하는 행위
*commit: add된 파일들을 스냅샷으로 남김

Git 로컬 저장소에 Commit 남기기#

  1. 임의의 파일 생성
  2. git 저장소의 상태 확인
    $ git status # untracked files 목록에 앞에서 생성한 파일명이 뜸
  3. commit에 반영할 파일 지정
    $ git add [파일명] # unstaged -> staged
    $ git status # changes to be committed 목록에 해당 파일이 new file로 존재함
  4. commit
    git commit -m "[커밋할 메시지]"
  5. commit 확인
    $ git log

2. Git의 Branch#

이슈가 발생했을 때, 이슈에 대한 코드를 복제하여 따로 가지치기하여 독립적인 환경에서 코드를 수정하며, 이때 메인 코드는 동시에 진행될 수 있다. 신규 코드를 추가하고자 할 때도 메인 코드는 그대로 진행하면서 새로운 가지치기를 통해 새로운 기능을 개발한다. 이후 작업이 완료되면 다시 합쳐서 최신 버전으로 만든다.

깃에는 기본적으로 master 브랜치가 생성되어 있다.

Git의 Branch 생성하기#

$ git branch -v # 브랜치 확인
$ git branch [브랜치명]

Git의 Branch 이름 바꾸기#

❗ Github의 정책이 바뀌어 master 브랜치의 이름을 main으로 바꿔야 코드 변경을 확인하기 쉽다.

$ git branch -M main

Git의 Branch 전환하기#

branch 전환

$ git checkout [브랜치명]

새로운 Branch에 변경 사항 적용#

파일 수정 후

git status # 수정한 파일이 변경되었다고(modified) 뜸
git add [파일명]
git commit -m [커밋 메시지]
git log # 로그의 브랜치들이 서로 다름, HEAD는 현재 작업 중인 브랜치를 의미
git checkout main # 브랜치를 다시 main 변경한 후 파일을 열어보면 다시 이전의 내용이 보임

Git의 Branch 병합하기#

$ git log # 현재 브랜치 확인
$ git checkout main
$ git merge [브랜치명]

Git의 Branch 삭제하기#

branch 삭제

$ git branch -d [브랜치명]

3. Git과 Github#

Github는 원격 저장소 중 가장 대중적이다.

원격 저장소 생성#

  1. 원격 저장소 생성
    Github 로그인 후 -> New -> 설정 후 -> Create Repository
  2. 로컬 저장소와 원격 저장소 연결
    $ git remote add [별칭] [원격저장소 주소] # 사용 중인 로컬 저장소에서 원격 저장소 지정
    *별칭은 origin을 주로 많이 사용
  3. 확인
    $ git remove -v
  4. 로컬 저장소에서 작업한 정보를 원격 저장소로 전달
    $ git push [원격저장소명] [브랜치명]

+ Github로 과제 제출하기#

  1. 과제가 있는 원격 저장소를 자신의 원격 저장소로 Fork한다.
  2. 자신의 로컬 저장소에서 브랜치를 만들어 작업을 진행하고 Push한다.
    $ git clone [원격저장소 주소] [폴더명]
    $ cd [폴더명]
    $ git branch [주차]/[이름] # ex) week3/이희주
    $ git checkout [주차]/[이름]
    *위의 폴더명에는 해당 원격 저장소 파일을 담을 폴더명을 적는다.
  3. 자신의 원격 저장소에서 과제 저장소로 Pull Request한다.
    Pull Request(PR)는 fork한 저장소에 수정한 사항을 반영하고 싶다고 요청을 보내는 것이다.
    기본적으로 fork한 저장소는 다른 사람의 저장소이기 때문에 함부로 push할 수 없다. 따라서 복제본을 만들어 원격 저장소로 가져왔고(fork), PR을 보냄으로써 협업을 완료할 수 있다.
