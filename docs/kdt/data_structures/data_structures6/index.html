1. 힙 (Heaps)#

= 이진 힙 (binary heap) 이진 트리의 한 종류로, root 노드가 항상 최댓값 또는 최솟값을 가지며 (최대 힙, 최소 힙), 완전 이진 트리다.

어떤 노드를 루트로 하더라도 서브트리는 모두 최대 힙 또는 최소 힙 형태를 보이므로 재귀적으로 정의 가능하다.

이진 탐색 트리와의 비교#

Q이진 탐색 트리힙
원소들은 완전히 크기 순으로 정렬되어 있는가?YN (느슨하게 연결)
특정 키 값을 가지는 원소를 빠르게 검색할 수 있는가?YN
부가의 제약 조건은 어떤 것인가?-완전 이진 트리여야 함

최대 힙 (Max Heap)의 추상적 자료구조#

연산 정의#

  1. 빈 최대 힙 생성: init()
  2. 새로운 원소 삽입: insert()
  3. 최대 원소 (root node) 제거(반환): remove()

데이터 표현#

배열로 최대 힙을 구현한다.

노드 번호 m을 기준으로 왼쪽 자식의 번호는 2 m, 오른쪽 자식의 번호는 2 m + 1, 부모 노드의 번호는 m // 2로 표현할 수 있다.

또한 힙은 완전 이진 트리이므로 노드의 추가와 삭제는 마지막 노드에서만 이루어진다.

배열로 최대 힙 구현#

1. 클래스 생성: MaxHeap#

class MaxHeap:
def __init__(self):
self.data = [None] # 0번 인덱스는 버림

2. 새로운 원소 삽입: insert()#

트리의 마지막 자리에 새로운 원소를 임시로 저장하고, 부모 노드와 키 값을 비교하여 값이 크다면 원소를 위로 이동시킨다.

원소의 개수가 n인 최대 힙에 새로운 원소를 삽입하는 경우, 부모 노드와의 대소 비교 최대 횟수를 따져 봤을 때 시간 복잡도는 O(log n)이다.

def insert(self, item):
self.data.append(item)
m = len(self.data) - 1
while m > 1
if self.data[m] > self.data[m//2]:
self.data[m], self.data[m//2] = self.data[m//2], self.
data[m]
m //= 2
else:
break

3. 최대 원소 삭제: remove()#

최댓값인 루트 노드를 제거하고, 트리 마지막 자리 노드를 임시로 루트 노드의 자리에 배치한 뒤 자식 노드와 키 값을 비교하여 값이 작다면 원소를 아래로 이동시키는데 이때 자식 노드가 둘이라면, 더 큰 값 쪽으로 옮긴다 (root에는 가장 큰 값이 와야 하기 때문).

원소의 개수가 n인 최대 힙에서 최대 원소를 삭제하는 경우, 자식 노드와의 대소 비교 최대 횟수는 2log n이므로 시간 복잡도는 O(log n)이다.

def remove(self):
if len(self.data) > 1:
self.data[1], self.data[-1] = self.data[-1], self.data[1]
data = self.data.pop()
self.maxHeapify(1)
else:
data = None
return data
def maxHeapify(self, i):
left = 2 * i
right = 2 * i + 1
# 자신(i), 왼쪽 자식(left), 오른쪽 자식(right) 중 최대를 찾음
if left < len(self.data) and self.data[left] > self.data[smallest]:
smallest = left
if right < len(self.data) and self.data[right] > self.data[smallest]:
smallest = right
if smallest != i:
self.data[i], self.data[smallest] = self.data[smallest], self.data[i]
self.maxHeapify(self, smallest)

최대/최소 힙의 응용#

  1. 우선순위 큐 (priority queue)
    Enqueue할 때, 느슨한 정렬을 이루고 있도록 한다. (O(log n))
    Dequeue할 때, 최댓값 또는 최솟값을 순서대로 추출한다. (O(log n))
    ❗ 양방향 연결 리스트를 이용한 구현과 효율성을 비교해보자.
  2. 힙 정렬 (heap sort)
    정렬되지 않은 원소들을 아무 순서로나 최대 힙에 삽입한다.(O(log n))
    삽입이 끝나면, 힙이 비어 있을 때까지 하나씩 삭제한다. (O(log n))
    원소들이 삭제된 순서가 원소들의 정렬 순서이며, 이때의 정렬 알고리즘의 시간 복잡도는 O(n log n)이다.

힙 정렬 (heap sort)의 구현#

def heapsort(unsorted):
H = MaxHeap()
for item in unsorted:
H.insert(item)
sorted = []
d = H.remove()
while d:
sorted.append(d)
d = H.remove()
return sorted

연습문제 - 더 맵게#

입출력 예#

scovilleKreturn
[1, 2, 3, 9, 10, 12]72

강의 풀이#

원칙: 앞 자리에 큰 수가 오는 것이 전체를 크게 만든다.

  1. number로부터 하나씩 꺼내어 모으되, 이미 모아둔 것 중 지금 등장한 것보다 작은 것들은 빼낸다.
  2. k개만큼 제거하면 남은 숫자는 모두 담는다.
  3. 아직 k 만큼 채우지 못한 경우 뒤에서 남은 수만큼 빼낸다.
from heapq import *
def solution(scoville, K):
answer = 0
heapify(scoville)
# 최악의 경우 O(n)
while True:
# O(log n)
min1 = heappop(scoville)
if min1 >= K:
break
if len(scoville) == 0:
answer = -1
break
# O(log n)
min2 = heappop(scoville)
new_scoville = min1 + 2 * min2
# O(log n)
heappush(scoville, new_scoville)
answer += 1
return answer

(힙을 사용하지 않은 경우)
시간 복잡도: O(n²)
최악의 경우: 수가 하나 남을 때까지 섞어야 하는 경우 (n-1회)
섞는 일에서 요구되는 계산량: O(n)

(힙을 사용한 경우)
시간 복잡도: O(n log n) 힙 구성: O(n log n)
삽입: O(log n)
삭제: O(log n)

내 풀이#

from heapq import *
def solution(scoville, K):
heapify(scoville)
answer = 0
while True:
a = heappop(scoville)
if a > K:
break
if not scoville:
answer = -1
