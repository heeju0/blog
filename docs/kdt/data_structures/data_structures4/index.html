큐

💡 『KDT 프로그래머스 인공지능 데브코스』에서 공부중..👀

1. 큐 (Queues)#

자료(data element)를 보관할 수 있는 선형 구조

선입선출(FIFO, First-In, First-Out)
-> 큐에서는 데이터를 넣을 때 한 쪽 끝에서 밀어 넣어야 하며(enqueue 연산), 꺼낼 때는 반대 쪽에서 뽑아야 한다(dequeue 연산).

큐의 동작#

초기 상태: 비어 있는 큐 (Q = Queue())
데이터 원소 A를 큐에 추가 (Q.enqueue(A))
데이터 원소 B를 큐에 추가 (Q.enqueue(B))
데이터 원소 꺼내기 (r1 = Q.dequeue(), r1에는 A가 담김)
데이터 원소 꺼내기 (r2 = Q.dequeue(), r2에는 B가 담김)

큐의 추상적 자료구조#

구현 방법#

방법1) 배열(array)을 이용하여 구현
-> python의 리스트와 메서드들을 이용한다.

방법2) 연결 리스트(linked list)를 이용하여 구현
-> 앞에서 구현한 양방향 연결 리스트를 이용한다.

연산 정의#

  1. 원소의 수: size()
    현재 큐에 들어 있는 데이터 원소의 수를 구한다.
  2. 큐가 비어 있는지 여부: isEmpty()
    현재 큐가 비어 있는지를 판단한다.
  3. 원소 추가: enqueue(x)
    데이터 원소 x를 큐에 추가한다.
  4. 원소 제거: dequeue()
    큐의 맨 앞에 저장된 데이터 원소를 리턴하고 제거한다.
  5. 원소 반환: peek() 큐의 맨 앞에 저장된 데이터 원소를 리턴한다.

배열로 스택 구현#

class ArrayQueue:
# 빈 큐 초기화
def __init__(self):
self.data = []
# 큐의 크기 리턴
def size(self):
return len(self.data)
# 큐가 비어 있는지 판단
def isEmpty(self):
return self.size() == 0
# 데이터 원소를 추가
def enqueue(self, item):
self.data.append(item)
# 데이터 원소 삭제 (리턴)
def dequeue(self):
return self.data.pop(0)
# 큐의 맨 앞 원소 리턴
def peak(self):
return self.data[0]

배열로 구현한 큐의 연산 복잡도#

연산복잡도
size()O(1)
isEmpty()O(1)
enqueue()O(1)
dequeue()O(n)
peek()O(1)

스택의 pop()은 맨 위의 값을 꺼내기 때문에 부담이 없으나, 큐의 dequeue()는 맨 앞의 원소를 꺼낸 뒤 모든 값이 한 칸씩 앞으로 이동하기 때문에 시간 복잡도가 O(n)이 된다.

양방향 연결 리스트로 스택 구현#

from doublylinkedlist import Node
from doublylinkedlist import DoublyLinkedList
class LinkedListQueue:
def __init__(self):
self.data = DoublyLinkedList()
def size(self):
return self.data.getLength()
def isEmpty(self):
return self.size() == 0
def enqueue(self, item):
node = Node(item)
self.data.insertAt(self.size()+1, node)
def dequeue(self):
return self.data.popAt(1)
def peek(self):
return self.data.getAt(1).data

큐의 활용#

(큐를 활용한 연습문제는 난이도가 높아 활용 예제로 대체)

  1. 자료를 생성하는 작업과 그 자료를 이용하는 작업이 비동기적으로 (asynchronously) 일어나는 경우 (Producer 1 -> Queue -> Consumer 1)
  2. 자료를 생성하는 작업이 여러 곳에서 일어나는 경우 (Producer 多 -> Queue -> Consumer 1)
  3. 자료를 생성하는 작업과 그 자료를 이용하는 작업이 양쪽 다 여러 곳에서 일어나는 경우 (Producer 多 -> Queue -> Consumer 多)
  4. 자료를 처리하여 새로운 자료를 생성하고, 나중에 그 자료를 또 처리해야 하는 작업의 경우

2. 환형 큐 (Circular Queues)#

운영체제와 같은 컴퓨터 시스템 내부에서 이용될 때는 자원이 무한정이 아니기 때문에 큐에 담을 수 있는 원소의 개수를 정해두고 큐를 구현하는 경우가 많다. 이 경우, 배열로 구현하게 되면 앞의 원소를 dequeue했을 때 큐의 길이에 비례하는 시간 복잡도를 가지기 때문에 환형 큐를 이용한다.

환형 큐는 정해진 개수의 저장 공간을 빙 돌려가며 이용하며, 데이터를 꺼내는 곳은 front, 데이터를 집어 넣는 곳은 rear라고 표현한다.
큐가 가득 차면 더 이상 원소를 넣을 수 없으므로 큐의 길이를 기억하고 있어야 한다.

환형 큐의 추상적 자료구조#

연산 정의#

  1. 원소의 수: size()
    현재 큐에 들어 있는 데이터 원소의 수를 구한다.
  2. 큐가 비어 있는지 여부: isEmpty()
    현재 큐가 비어 있는지를 판단한다.
  3. 큐가 꽉 찼는지 여부: isFull()
    큐에 데이터 원소가 꽉 차 있는지를 판단한다.
  4. 원소 추가: enqueue(x)
    데이터 원소 x를 큐에 추가한다.
  5. 원소 제거: dequeue()
    큐의 맨 앞에 저장된 데이터 원소를 리턴하고 제거한다.
  6. 원소 반환: peak() 큐의 맨 앞에 저장된 데이터 원소를 리턴한다.

배열로 환형 큐 구현#

정해진 길이 n의 리스트를 확보해 두고 시작하고, 0번 인덱스에 값을 넣는다. 큐에 값이 enqueue되면, 그 값이 rear가 된다. dequeue할 때는 0번부터 front가 되며, 이 값을 꺼낸다. front와 rear을 적절히 계산하여 배열을 환형으로 재활용해야 한다.

class CircularQueue:
def __init__(self, n):
self.maxCount = n
self.data = [None] * n
self.count = 0 # 현재 큐에 들어 있는 원소의 수
self.front = -1
self.rear = -1
def size(self):
return self.count
def isEmpty(self):
return self.count == 0
def isFull(self):
return self.count == self.maxCount
def enqueue(self, x):
if self.isFull():
raise IndexError('Queue full')
self.rear = (self.rear + 1)%self.maxCount
self.data[self.rear] = x
self.count += 1
def dequeue(self):
if self.isEmpty():
raise IndexError('Queue empty')
self.front = (self.front + 1)%self.maxCount
x = self.data[self.front]
self.count -= 1
return x
def peak(self):
if self.isEmpty():
raise IndexError('Queue empty')
return self.data[(self.front + 1)%self.maxCount]

3. 우선순위 큐 (Priority Queues)#

큐가 FIFO(First-In First-Out) 방식을 따르지 않고 원소들의 우선순위에 따라 큐에서 빠져나오는 방식

ex) 작은 수의 우선순위가 더 높다고 할 때,
Enqueue: 6 -> 7 -> 3 -> 2
Dequeue: 2 -> 3 -> 6 -> 7

우선순위 큐의 활용#

운영체제의 CPU 스케줄러

우선순위 큐의 추상적 자료구조#

구현 방법#

방식1) Enqueue 할 때 우선순위 순서 유지
방식2) Dequeue 할 때 우선순위가 높은 것 선택

방식1이 조금 더 유리하다. 값이 우선순위에 상관 없이 늘어져 있다면 Dequeue 할 때 모든 데이터를 확인해야 하므로 시간 복잡도가 O(n)이다.

방법1) 선형 배열을 이용하여 구현
방법2) 연결 리스트를 이용하여 구현

방법2가 더 유리하다. 선형 배열의 경우 중간에 원소를 끼워 넣기가 어렵다.
그러나 공간 복잡도에 있어서는 선형 배열이 더 낫다.

양방향 연결 리스트로 우선순위 큐 구현#

from doublylinkedlist import Node, DoublyLinkedList
class PriorityQueue:
def __init__(self, x):
self.queue = DoublyLinkedList()
def enqueue(self, x):
newNode = Node(x)
curr = self.queue.head
# getAt() 메서드는 while문 반복 시마다 위치를 다시 찾아가기 때문에 사용하지 않음
while curr.next.data != None and x < curr.next.data:
curr = curr.next
self.queue.insertAfter(curr, newNode)
