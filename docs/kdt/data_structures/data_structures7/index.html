해시

💡 『KDT 프로그래머스 인공지능 데브코스』에서 공부중..👀

해시 (Hash)#

실제 데이터 (키, key)의 위치를 정해 해시 테이블 (hash table) 내에 저장하며, 해시 테이블 안의 칸들을 해시 버킷 (hash bucket)이라고 한다. 이때, 해당 키를 해시 테이블의 어떤 칸에 저장할지 결정하기 위해 해시 함수 (hash function)을 이용하며, 해시 함수는 해시 버킷의 갯수에 맞게 구성되어야 한다.

해시 충돌 (hash collision)#

서로 다른 키가 같은 해시 버킷에 해시되는 현상

연습문제 - 완주하지 못한 선수#

입출력 예#

participantcompletionreturn
["leo", "kiki", "eden"]["eden", "kiki"]"leo"
["marina", "josipa", "nikola", "vinko", "filipa"]["josipa", "filipa", "marina", "nikola"]"vinko"
["mislav", "stanko", "mislav", "ana"]["stanko", "ana", "mislav"]"mislav"

강의 풀이 2#

  1. 먼저 participant에서 등장하는 이름의 횟수를 센다.
  2. completion에서 등장하는 이름의 횟수를 뺀다.
  3. 참가하지 않은 선수의 이름을 리턴한다.
def solution(participant, completion):
d = {}
# 이름 수 세기 - O(n)
for x in participant:
d[x] = d.get(x, 0) + 1
# 참가한 선수의 이름 제거 - O(n)
for x in completion:
d[x] -= 1
# 참가하지 않은 선수 리스트 - O(n)
dnf = [k for k, v in d.items() if v > 0]
answer = dnf[0]
return answer

시간 복잡도:
participant의 길이를 n이라 하면, O(n)의 시간 복잡도를 가진다.

강의 풀이 1#

  1. participant와 completion을 오름차순 정렬한다.
  2. 인덱스 순서대로 순환하면 participant와 completion의 값이 다르면 리턴한다.
def solution(participant, completion):
# 정렬 - O(n log n)
participant.sort()
completion.sort()
# participant와 completion의 원소 비교 - O(n)
for i in range(len(completion)):
if participant[i] != completion[i]:
return participant[i]
return participant[-1]

시간 복잡도:
participant의 길이를 n이라 하면, O(n log n)의 시간 복잡도를 가진다.
(해시 방법보다 시간 복잡도가 더 크다.)

내 풀이#

from collections import defaultdict
def solution(participant, completion):
dic = defaultdict(int)
for i in participant:
dic[i] += 1
for i in completion:
dic[i] -= 1
for i in dic:
if dic[i] == 1:
return i
