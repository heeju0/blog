1. 선형 배열 (Linear Arrays)#

데이터가 일렬로 늘어선 형태

1-1. 배열 (Array)#

파이썬에서는 리스트(List)를 사용하여 배열을 나타낼 수 있으며, 다른 언어들과는 다르게 데이터들이 같은 타입이 아니어도 입력 가능하다.

리스트 (배열) 연산#

1. 원소 추가 및 꺼내기#

원소 덧붙이기: List객체.append(원소)
끝에서 꺼내기: List객체.pop()
-> 배열의 끝에 원소를 덧붙이는 연산과 끝의 원소를 꺼내는 연산은 리스트의 길이와 무관하게 시간 복잡도가 O(1)이다.

2. 원소 삽입 및 삭제#

원소 삽입하기: List객체.insert(idx, 원소)
원소 삭제하기: del(List객체[idx])
-> 리스트의 길이가 길수록 오래 걸리며, 시간 복잡도가 O(n)이다. 이와 같이 리스트의 길이에 비례한 시간을 선형 시간이라고 표현한다.

💡 del()과 pop()의 차이
pop(): 해당 원소를 꺼내는 개념. pop() 실행 시 해당 원소가 출력되며, 리스트에서는 해당 원소가 지워진다.
del(): 해당 원소를 제거하는 개념. del() 실행 시 출력값 없이 해당 원소가 지워진다.

3. 원소 탐색#

List객체.index(원소)
-> 시간 복잡도가 O(1)이다. (왜..?)

연습문제#

Hint)
주어진 원소를 삽입할 위치를 찾는다.
해당 위치에 원소를 삽입한다.
-> 그 결과는 여전히 정렬된 상태를 유지하는 리스트

(1) 리스트에 새로운 요소 삽입하기
def solution(L, x):
if x < L[0]:
L.insert(0, x)
elif x > L[-1]:
L.append(x)
else:
for i in range(len(L)-1):
if L[i] <= x < L[i+1]:
L.insert(i+1, x)
break
return L

Hint)
index() 메서드를 이용한다.
리스트 슬라이싱을 이용해서 반복하여 찾는다.

(2) 리스트에서 원소 찾아내기
def solution(L, x):
if x in L:
answer = [i for i, l in enumerate(L) if l == x]
else:
answer = [-1]
return answer

2. 정렬과 탐색#

2-1. 정렬 (Sort)#

배열 안의 원소들을 정해진 규칙에 따라 새로 늘어놓는 것
문자열로 이루어진 리스트의 경우 정렬은 알파벳 순서를 따른다.

Python 리스트의 정렬#

1. sorted(List객체)#

내장 함수
정렬된 새로운 리스트를 리턴한다.

2. List객체.sort()#

리스트의 메서드로, 해당 리스트를 정렬한다.

3. 정렬의 순서 변경#

reverse=True 옵션 사용

4. 정렬 기준 설정#

key 옵션 사용

예제#

예제1) 리스트 L을 문자열의 길이 순서대로 정렬
L = ['abcd', 'xyz', 'spam']
sorted(L, key=labmda x: len(x))

❗ 문자열의 길이가 동일한 경우 인덱스가 작은 것부터 나열됨

예제2) 리스트 L을 name을 기준으로 정렬
L = [{'name':'John', 'score':83}, {'name':'Paul', 'score':92}]
L.sort(key=lambda x: x['name'])
예제3) 리스트 L을 점수가 높은 순으로 정렬
L = [{'name':'John', 'score':83}, {'name':'Paul', 'score':92}]
L.sort(key=lambda x: x['score'], reverse=True)

2-2. 탐색 (Search)#

탐색 알고리즘#

1. 선형 탐색 (Linear Search)#

앞에서 뒤로 이동하며 원하는 값을 찾을 때까지 진행하는 방법
정렬의 성질을 전혀 이용하지 않는다.
시간 복잡도는 O(n)으로, 최악의 경우 모든 원소를 다 비교해봐야 한다.

def linear_search(L, x):
i = 0
while i < len(L) and L[i] != x:
i += 1
if i < len(L):
return i
else:
return -1

2. 이진 탐색 (Binary Search)#

리스트의 중앙값을 기준으로 자르고 원하는 값이 포함된 리스트를 선택하여 원하는 값을 찾을 때까지 반복하는 방법
리스트가 이미 정렬되어 있는 경우에만 사용 가능하다.
한 번 비교할 때마다 리스트를 반으로 줄이므로 시간 복잡도는 O(log n)이다.

(03) 이진탐색
def binary_search(L, x):
lower = 0
upper = len(L) - 1
idx = -1
while lower <= upper:
mid = (lower + upper)//2
if L[mid] == x:
return mid
elif L[mid] < x:
lower = x + 1
else:
upper = x - 1
return -1

3. 재귀 알고리즘 (Recursive Algorithms)#

3-1. 재귀 함수 (Recursive Functions)#

하나의 함수에서 자신을 다시 호출하여 작업을 수행하는 것

ex) 이진 트리 (Binary Trees)
한 노드를 기준으로 이진 트리에서 왼쪽 서브 트리의 노드들은 모두 작거나 같고, 오른쪽 서브 트리의 노드들은 모두 크도록 요구하는 자료 구조가 정의되어 있다고 할 때, 이 원칙을 모든 노드에 대해 재귀적으로 적용할 수 있으며, 또한 찾고자 하는 값도 재귀 함수를 통해 찾을 수 있다.

예제#

예제1) 1부터 n까지 모든 자연수의 합
def sum(n):
if n > 0:
return n + sum(n-1)
else:
return n

❗ 재귀 함수에서는 종결 조건이 반드시 필요하다는 것을 유의해야 한다.

예제2) n! 구현
def factorial(n):
if n > 1:
return n * factorial(n-1)
else:
return 1

재귀 함수의 효율#

Recursive Version와 Iterative Version의 시간 복잡도는 모두 시간 복잡도가 O(n)이다.
그러나 효율성 측면에서 보면 Recursive Ver.은 n이 증가할수록 함수를 호출하고 리턴하는 부가적인 작업이 필요하기 때문에 Recursive Ver.의 효율이 더 떨어진다고 볼 수 있다.

Recursive Ver.
def sum(n):
if n > 0:
return n + sum(n-1)
else:
return n

vs.

Iterative Ver.
def sum(n):
s = 0
while n > 0:
s += n
n -= 1
return s

vs.

수학적인 방법 (O(1))
def sum(n):
return n * (n + 1) // 2

연습문제#

Fibonacci 순열 구현하기

Hint)
F₀ = 0
F₁ = 1
Fₙ = Fₙ₋₁ - Fₙ₋₂

(04) 피보나치 순열 - Recursive Ver.
def rec(n):
if n == 0 or n == 1:
return n
else:
return fibonacci(n-1) + fibonacci(n-2)
(04) 피보나치 순열 - Iterative Ver.
def iter(n):
s = 0
f1, f2 = 0, 1
if n == 1:
return 1
while n > 1:
s = f1 + f2
f1, f2 = f2, s
n -= 1
return s
(추가) Iterative Ver.과 Recursive Ver.의 시간 비교
import time
while True:
nbr = int(input('Enter a number: '))
if nbr == -1:
break
ts = time.time()
fibo = iter(nbr)
ts = time.time() - ts
print('Iterative Version: %d (%.3f)'%(fibo, ts))
ts = time.time()
fibo = rec(nbr)
ts = time.time() - ts
print('Recursive Version: %d (%.3f)'%(fibo, ts))

3-2. 재귀 알고리즘 응용#

예제#

예제1) n개의 서로 다른 원소에서 m개를 택하는 경우의 수 구현

math 라이브러리 사용
from math import factorial as f
def combi(n, m):
return f(n) / (f(m) * f(n-m))

Recursive Ver.
def combi(n, m):
if n == m or m == 0:
return 1
return combi(n-1, m) + combi(n-1, m-1)

예제2) 하노이의 탑
크기 순서대로 쌓여 있는 한 기둥의 원반들을 모두 다른 기둥으로 옮긴다. 이때, 큰 원반을 작은 원반 위로 올릴 수 없다.

재귀 알고리즘의 효율#

fibonacci 수열을 예로 들면, fibo(4) = fibo(3) + fibo(2)에서
fibo(3) = fibo(2) + fibo(1),
fibo(2) = fibo(1) + fibo(0)
이 된다. 이 과정에서 fibo(2)와 fibo(1)은 중복해서 호출된다는 문제점이 있다.

연습문제#

(05) 재귀적 이진 탐색 구현
def binsearch(L, x, lower, upper):
if lower > upper:
return -1
mid = (lower + upper)//2
if x == L[mid]:
return mid
if x < L[mid]:
return binsearch(L, x, lower, mid-1)
else:
return binsearch(L, x, mid+1, upper)

4. 정렬 알고리즘 (Sort Algorithm)#

4-1. 삽입 정렬 (Insertion Sort)#

배열의 모든 요소를 앞에서부터 차례대로 이미 정렬된 부분과 비교하여 자신의 위치를 찾아 삽입하는 과정을 반복하는 정렬 방법이다.
삽입 정렬의 시간 복잡도는 O(n²)이다.

4-2. 병합 정렬 (Merge Sort)#

정렬할 데이터를 반복하여 반으로 나누어 각각을 정렬하고 (O(log n)), 정렬된 데이터를 두 묶음씩 합치는 정렬하는 방법이다 (O(n)).
병합 정렬의 시간 복잡도는 O(n log n)이다.

5. 알고리즘의 복잡도 (Complexity of Algorithms)#

시간 복잡도 (Time Complexity): 문제의 크기와 이를 해결하는 데 걸리는 시간 사이의 관계
공간 복잡도 (Space Complexity): 문제의 크기와 이를 해결하는 데 필요한 메모리 공간 사이의 관계

5-1. 시간 복잡도#

1. 평균 시간 복잡도 (Average Time Complexity)#

임의의 입력 패턴을 가정했을 때 소요되는 시간의 평균

2. 최악 시간 복잡도 (Worst-case Time Complexity)#

가장 긴 시간을 소요하게 만드는 입력에 따라 소요되는 시간

5-2. Big-O Notation#

점근 표기법 (Asymptotic Notation)의 하나로, 어떤 함수의 증가 양상을 다른 함수와의 비교로 표현하는 방식이다.
알고리즘의 복잡도를 표현할 때 가장 보편적으로 사용하며, 계수는 중요하지 않다.

입력의 크기가 n일 때,
O(log n): 입력의 크기의 로그에 비례하는 시간이 소요된다.
O(n): 입력의 크기에 비례하는 시간이 소요된다.

AlgorithmBest caseAverage caseWorst caseExample
선형 시간 알고리즘O(n)O(n)선형 탐색 알고리즘
로그 시간 알고리즘O(log n)O(n)이진 탐색 알고리즘
이차 시간 알고리즘O(n)O(n²)삽입 정렬 (Insertion Sort)

cf> 삽입 정렬보다 낮은 복잡도를 갖는 정렬 알고리즘
ex) 병합 정렬 (Merge Sort): O(n log n)
참고: 입력 패턴에 따라 정렬 속도에 차이가 있지만, 정렬 문제에 있어 O(n log n)보다 낮은 복잡도를 갖는 알고리즘은 존재할 수 없음이 증명되어 있다.
