연결 리스트 (Linked Lists)#

연결 리스트는 선형 배열과 구조가 유사하다.

연결 리스트에서 알아야 할 요소들#

Head: 리스트의 맨 앞
Tail: 리스트의 맨 끝
# of nodes: 노드의 개수

이 요소들과 리스트가 가질 수 있는 연산들을 저장한 것이 이 리스트의 추상적 자료구조가 된다.

배열 vs. 연결 리스트#

ㅤ배열리스트
저장 공간연속한 위치임의의 위치
특정 원소 지칭매우 간편 (인덱스)선형탐색과 유사 (k번째 값을 찾음)
시간 복잡도O(1)O(n)

연결 리스트의 추상적 자료구조#

추상적 자료구조 (Abstract Data Structures)#

자료구조의 내부 구현은 숨기고, 외부에서 보이는 것(데이터, 연산)을 제공하는 구조

Data: 정수, 문자열, 레코드 등
A set of operations: 삽입, 삭제, 순회, 정렬, 탐색 등

연산 정의#

1) 특정 원소 참조 (k번째)
2) 리스트 순회 차례로 노드들을 방문하는 연산 3) 길이 얻어내기
4) 원소 삽입
5) 원소 삭제
6) 두 리스트 연결

기본적인 연결 리스트 구현#


앞의 화살표가 뒤의 값을 가리키도록 구성된 리스트 형태
각 아이템을 노드라고 하며, Node 하나에는 Data 다음 노드를 가리키는 Link 가지고 있다.

(Node 내의 데이터는 문자열, 레코드, 또 다른 연결 리스트 등 다른 구조로 이루어질 수 있으나, 여기서는 간단하게 숫자 데이터에 대해서만 다룬다.)

1. 클래스 생성: Node, LinkedList#

Node 클래스: 데이터(data), 링크(next)

class Node:
def __init__(self, item):
self.data = item
self.next = None # 아직 아무 값도 가리키고 있지 않음

LinkedList 클래스: 노드의 개수(nodeCount), 맨 앞(head), 맨 끝(tail)

class LinkedList:
def __init__(self):
self.nodeCount = 0
self.head = None # 아직 아무 값도 가리키고 있지 않음
self.tail = None # 아직 아무 값도 가리키고 있지 않음

2. 특정 원소 참조: getAt()#

원소의 순서는 1부터 시작하도록 지정한다. (0번은 다른 목적으로 사용하고자 한다.)

def getAt(self, pos):
if pos < 1 or pos > self.nodeCount:
return None
i = 1
curr = self.head
while i < pos:
curr = curr.next
i += 1
return curr

3. 리스트 순회: traverse()#

def traverse(self):
lst = []
curr = self.head
while curr:
lst.append(curr.data)
curr = curr.next
return lst

4. 길이 얻어내기: getLength()#

nodeCount를 리턴하는 단순한 과정

def getLength(self):
return self.nodeCount

5. 원소 삽입: insertAt()#

pos가 가리키는 위치에 newNode를 삽입하고, 성공이면 True, 실패이면 False 리턴
삽입하고자 하는 위치가 리스트의 맨 앞인 경우에는, prev가 없으며 head를 조정해야 할 필요가 있다.
삽입하고자 하는 위치가 리스트의 맨 끝인 경우에는 tail을 조정해야 할 필요가 있다. 또한, 이 경우 맨 앞에서부터 찾아갈 필요가 없이 바로 tail을 찾을 수 있다.

def insertAt(self, pos, newNode):
if pos < 1 or pos > self.nodeCound + 1:
return False
# 리스트의 맨 앞인 경우
if pos == 1:
newNode.next = self.head
self.head = newNode
else:
# 리스트의 맨 끝인 경우 getAt()을 사용하지 않고 바로 tail을 적용
if pos == self.nodeCount + 1:
prev = self.tail
# 중간인 경우
else:
prev = self.getAt(pos-1)
newNode.next = prev.next
prev.next = newNode
# 리스트의 맨 끝인 경우
if pos == self.nodeCount + 1:
self.tail = newNode
self.nodeCount += 1
return True

원소 삽입의 시간 복잡도는
맨 앞에 삽입하는 경우 $O(1)$
중간에 삽입하는 경우 $O(n)$ (평균적으로 리스트의 길이가 길어질수록 복잡도가 커짐)
맨 끝에 삽입하는 경우 $O(1)$

6. 원소 삭제: popAt()#

pos가 가리키는 위치의 node를 삭제하고, 그 node의 데이터를 리턴
삭제하고자 하는 노드가 맨 앞에 있을 때는 prev가 없으며 head를 조정해야 한다.
삭제하고자 하는 노드가 맨 뒤에 있을 때는 tail을 조정해야 한다.

def popAt(self, pos):
if pos < 1 or pos > self.nodeCount:
raise IndexError
curr = self.getAt(pos)
# 노드가 하나인 경우
if self.nodeCount == 1:
self.head = None
self.tail = None
# 리스트의 맨 앞인 경우
elif pos == 1:
self.head = curr.next
else:
prev = self.getAt(pos-1)
# 리스트의 맨 뒤인 경우
if pos == self.nodeCount:
prev.next = None
self.tail = prev
# 중간인 경우
else:
prev.next = curr.next
self.nodeCount -= 1
return curr.data

원소 삭제의 시간 복잡도는
맨 앞에서 삭제하는 경우 O(1)
중간에서 삭제하는 경우 O(n)
맨 뒤에서 삭제하는 경우 O(n)

7. 두 리스트 연결: concat()#

def concat(self, L):
self.tail.next = L.head
# L이 []인 경우를 고려해야 함
if L.tail:
self.tail = L.tail
self.nodeCount += L.nodeCount

self.next = None

def __init__(self):
self.nodeCount = 0
self.head = None
self.tail = None
def __repr__(self):
if self.nodeCount == 0:
return 'LinkedList: empty'
s = ''
curr = self.head
while curr is not None:
s += repr(curr.data)
if curr.next is not None:
s += ' -> '
curr = curr.next
return s
# 특정 원소 참조

if pos < 1 or pos > self.nodeCount:
return None
i = 1
curr = self.head

curr = curr.next
i += 1
return curr
# 리스트 순회

return result

연결 리스트 개선#

스마트폰에서 실행 중인 어플 목록을 보거나 특정 어플들을 삭제할 때와 같이 삽입과 삭제가 유연하다는 것이 큰 장점이다.

그러나 위에서 구현한 원소 삽입 메서드는 맨 앞에서부터 n번째 노드를 찾아가야 하기 때문에 보다 효율적인 코드가 필요하다. -> insertAfter(prev, newNode), popAfter(prev) 메서드 생성

위의 메서드들은 맨 앞에서 삭제와 추가를 하기 어렵기 때문에 이런 문제점을 해결하고자 맨 앞에 dummy node (아무런 기능이 없는 노드)를 추가한다. 이때, dummy node에 0번 인덱스를 부여한다.

1. dummy node 추가#

class LinkedList:
def __init__(self):
self.nodeCount = 0
self.head = Node(None) # dummy node 생성
self.tail = None
self.head.next = self.tail

2. 리스트 순회: traverse()#

def traverse(self):
result = []
curr = self.head
while curr.next:
# head에는 데이터가 없기 때문에 먼저 next를 curr로 가져오고 result에 추가
curr = curr.next
result.append(curr.data)
return result

3. 특정 원소 참조: getAt()#

def getAt(self, pos):
# 0번에 dummy node가 추가되었으므로 0번을 기준으로 적용
if pos < 0 or pos > self.nodeCount:
return None
i = 0
curr = self.head
while i < pos:
curr = curr.next
i += 1
return curr

4. 원소 삽입: insertAfter(), insertAt()#

insertAfter(): prev가 가리키는 node의 다음에 newNode를 삽입하고, 성공하면 True, 실패하면 False를 리턴한다.

def insertAfter(self, prev, newNode):
newNode.next = prev.next
if prev.next is None:
self.tail = newNode
prev.next = newNode
self.nodeCount += 1
return True
def insertAt(self, pos, newNode):
if pos < 1 or pos > self.nodeCount + 1:
return False
if pos != 1 and pos == self.nodeCount + 1:
prev = self.tail
else:
prev = self.getAt(pos-1)
return self.insertAfter(prev, newNode)

5. 원소 삭제: popAfter(), popAt()#

popAfter(): prev의 다음 node를 삭제하고, 그 node의 data를 리턴한다.
prev가 마지막 node일 때는 None을 리턴하며, 리스트의 맨 끝 node를 삭제할 때는 tail을 조정해야 한다.

def popAfter(self, prev):
if prev.next is None:
return None
curr = prev.next
prev.next = curr.next
if curr.next is None:
self.tail = prev
self.nodeCount -= 1
return curr.data
def popAt(self, pos):
if pos < 1 or pos > self.nodeCount:
raise IndexError
prev = self.getAt(pos-1)
return self.popAfter(prev)

6. 두 리스트 합치기: concat()#

두 번째 리스트의 head 바로 다음 데이터와 연결한다.

def concat(self, L):
self.tail.next = L.head.next
if L.tail:
self.tail = L.tail
self.nodeCount += L.nodeCount

양방향 연결 리스트 (Doubly Linked Lists) 구현#

앞서 구현한 연결 리스트는 next를 통해 전진은 가능하지만 후진은 불가능한 구조다. 양방향 연결 리스트는 양쪽 방향으로 링크를 연결하여 다음 node뿐만 아니라 이전 node로도 진행이 가능하다.

1. 두 개의 클래스 생성: Node, DoublyLinkedList#

Node 클래스에 prev 변수를 생성하고, 리스트의 처음과 끝에 dummy node를 추가하도록 LinkedList를 작성한다.

class Node:
def __init__(self, item):
self.data = item
self.prev = None
self.next = None
class DoublyLinkedList:
def __init__(self):
self.nodeCount = 0
self.head = Node(None) # 맨 앞에 dummy node 생성
self.tail = Node(None) # 맨 뒤에 dummy node 생성
self.head.prev = None
self.head.next = self.tail
self.tail.prev = self.head
self.tail.next = None

2. 리스트 순회: traverse()#

def traverse(self):
result = []
curr = self.head
while curr.next.next:
curr = curr.next
result.append(curr.data)
return result

3. 리스트 역순회: reverse()#

def reverse(self):
result = []
curr = self.tail
while curr.prev.prev:
curr = curr.prev
result.append(curr.data)
return result

4. 특정 원소 참조: getAt()#

def getAt(self, pos):
if pos < 0 or pos > self.nodeCount:
return None
i = 0
if pos > self.nodeCount//2:
curr = self.tail
while i < self.nodeCount - pos + 1:
curr = curr.prev
i += 1
else:
curr = self.head
while i < pos:
curr = curr.next
i += 1
return curr

5. 원소 삽입: insertBefore(), insertAfter(), insertAt()#

def insertBefore(self, next, newNode):
prev = next.prev
newNode.prev = prev
newNode.next = next
prev.next = newNode
next.prev = newNode
self.nodeCount += 1
return True
def insertAfter(self, prev, newNode):
next = prev.next
newNode.prev = prev
newNode.next = next
prev.next = newNode
next.prev = newNode
self.nodeCount += 1
return True
def insertAt(self, pos, newNode):
if pos < 1 or pos > self.nodeCount + 1:
return False
prev = self.getAt(pos-1)
return self.insertAfter(prev, newNode)

6. 원소 삭제: popBefore(), popAfter(), popAt()#

def popBefore(self, next):
curr = next.prev
next.prev = curr.prev
curr.prev.next = next
self.nodeCount -= 1
return curr.data
def popAfter(self, prev):
curr = prev.next
prev.next = curr.next
curr.next.prev = prev
self.nodeCount -= 1
return curr.data
def popAt(self, pos):
if pos < 1 or pos > self.nodeCount:
raise IndexError
prev = self.getAt(pos-1)
return self.popAfter(prev)

7. 두 리스트 연결: concat()#

❗ L.head.next.prev = self.tail.prev 이 부분이 자꾸 헷갈린다.. 주의!

def concat(self, L):
self.tail.prev.next = L.head.next
L.head.next.prev = self.tail.prev
self.tail = L.tail
self.nodeCount += L.nodeCount
