트리

💡 『KDT 프로그래머스 인공지능 데브코스』에서 공부중..👀

1. 트리 (Trees)#

정점(node)와 간선(edge)을 이용하여 데이터의 배치 형태를 추상화한 자료 구조

용어 정리#

그래프 (graphs)
정점 (vetex, node), 간선 (edge, link)
유향 (directed) 그래프, 무향 (undirected) 그래프

스택 (stack)
큐 (queue)

부모 (Parent) 노드와 자식 (Child) 노드#

간선으로 연결된 두 개의 노드 중 root 쪽에 가까운 노드를 부모 노드, leaf 쪽에 가까운 노드를 자식 노드라고 하며, 같은 부모 노드 아래에 달려 있는 자식 노드들을 형제 (sibling) 노드라고 한다.

한 노드가 root 쪽으로 올라가면서 만나는 모든 노드는 조상 (ancestor) 노드, leaf 쪽으로 내려가면서 만나는 모든 노드는 후손 (descendant) 노드라고 한다.

노드의 수준 (Level)#

root 노드의 level은 0이며, 아래로 내려갈 수록 레벨이 올라감

트리의 높이 (Height)#

= 트리의 깊이 (Depth)
트리의 높이 (height) = 최대 수준 (level) + 1

부분 트리 (서브 트리, Subtree)#

어떤 노드를 기준으로 그 노드와 하위 노드들로 이루어진 트리

노드의 차수 (Degree)#

= 각 노드의 자식 (서브트리)의 수

2. 이진 트리 (Binary Tree)#

모든 노드의 차수가 2 이하인 트리

이진 트리는 루트 노드 + 왼쪽 서브트리 + 오른쪽 서브트리로 이루어지며, 이때, 서브트리들이 모두 이진 트리이거나 빈 트리 (empty tree)인 경우 이진 트리를 재귀적으로 정의할 수 있다.

포화 이진 트리 (Full Binary Tree)#

모든 레벨에서 노드들이 모두 채워져 있는 이진 트리
높이가 k이고 노드의 개수가 (2^k - 1)인 이진 트리

완전 이진 트리 (Complete Binary Tree)#

높이가 k인 완전 이진 트리는 레벨 k - 2 까지는 모든 노드가 2개의 자식을 가진 포화 이진 트리 형태이며, 레벨 k - 1에서는 왼쪽부터 노드가 순차적으로 채워져 있는 이진 트리를 말한다.

이진 트리의 추상적 자료구조#

연산 정의#

  1. 노드의 수: size()
    현재 트리에 포함되어 있는 노드의 수를 구한다.
  2. 트리의 깊이: depth()
    현재 트리의 깊이 (또는 높이) 를 구한다.
  3. 순회 (traversal)
    정해진 순서에 따라 노드들을 순회한다.

이진 트리의 구현#

1. 클래스 생성: Node, Tree#

노드에는 데이터와 left child, right child가 있다.

class Node:
def __init__(self, item):
self.data = item
self.left = None
self.right = None

root 노드만 가리킨다면 left child와 right child는 간선으로 연결되어 있기 때문에 모두 찾아갈 수 있다.

class BinaryTree:
def __init__(self, r):
self.root = r

2. 노드의 수: size()#

트리는 정의 부터 재귀적이기 때문에 노드의 수 또한 재귀적인 방법으로 쉽게 구할 수 있다.

각각의 노드를 root로 하는 서브트리의 size도 구할 수 있어야 하기 때문에 BinaryTree 클래스의 메서드 뿐만 아니라 Node 클래스에서 봤을 때 자기 자신이 root인 서브트리의 사이즈를 구하는 멤버 메서드도 제공해야 한다.

class Node:
def size(self):
# 왼쪽 서브트리의 size
l = self.left.size() if self.left else 0
# 오른쪽 서브트리의 size
r = self.right.size() if self.right else 0
# 전체 size 리턴
return l + r + 1
class BinaryTree:
def size(self):
# 빈 트리인지 판별
if self.root:
return self.root.size()
else:
return 0

3. 트리의 깊이: depth()#

마찬가지로 재귀적인 방법으로 쉽게 구할 수 있다.

class Node:
def depth(self):
l = self.left.depth() if self.left else 0
r = self.right.depth() if self.right else 0
return max(l, r) + 1
class BinaryTree:
def depth(self):
if self.root:
return self.root.depth()
else:
return []

4. 순회 (traversal)#

깊이 우선 순회 (Depth First Traversal)#

중위 순회 (in-order traversal): left subtree -> 자기 자신 -> right subtree 순으로 순회한다.

class Node:
def inorder(self):
traversal = []
# 왼쪽 서브트리가 있다면 추가
if self.left:
traversal += self.left.inorder()
# 자기 자신 추가
traversal.append(self.data)
# 오른쪽 서브트리가 있다면 추가
if self.right:
traversal += self.right.inorder()
return traversal
class BinaryTree:
def inorder(self):
if self.root:
return self.root.inorder()
else:
return []

전위 순회 (pre-order traversal): 자기 자신 -> left subtree -> right subtree 순으로 순회한다.

class Node:
def preorder(self):
traversal = [self.data]
if self.left:
traversal += self.left.preorder()
if self.right:
traversal += self.right.preorder()
return traversal
class BinaryTree:
def preorder(self):
if self.root:
return self.root.preorder()
else:
return []

후위 순회 (post-order traversal): left subtree -> right subtree -> 자기 자신 순으로 순회한다.

class Node:
def postorder(self):
traversal = []
if self.left:
traversal += self.left.postorder()
if self.right:
traversal += self.right.postorder()
traversal.append(self.data)
return traversal
class BinaryTree:
def postorder(self):
if self.root:
return self.root.traversal
else:
return 0

넓이 우선 순회 (Breadth First Traversal)#

수준(level)이 낮은 노드를 우선으로 방문하며, 같은 수준의 노드들 사이에는 부모 노드의 방문 순서에 따라 왼쪽 자식 노드를 오른쪽 자식 노드보다 먼저 방문한다.

한 노드를 방문했을 때, 나중에 방문할 노드들을 순서대로 기록해 두어야 하는데 이때 큐(queue)를 이용한다.

넓이 우선 순회는 재귀적인 방법이 적합하지 않다. 따라서 BinaryTree에 구현하면 된다.

class BinaryTree:
def bfs(self):
# 초기화
traversal = []
q = ArrayQueue()
# 빈 트리가 아니면 root node를 큐에 추가 (enqueue)
if self.root:
q.enqueue(self.root)
# q가 비어 있지 않은 동안
while q:
# q에서 원소 추출
node = q.dequeue()
# node 방문
traversal.append(node.data)
# q에 자식 노드 추가
if node.left:
q.enqueue(node.left)
if node.right:
q.enqueue(node.right)
return traversal

3. 이진 탐색 트리 (Binary Search Trees)#

모든 노드에 대해서,
왼쪽 서브트리에 있는 데이터는 모두 현재 노드의 값보다 작고
오른쪽 서브트리에 있는 데이터는 모두 현재 노드의 값보다 큰
성질을 만족하는 이진 트리

(여기서 중복되는 데이터 원소는 없는 것으로 가정한다.)

정렬된 배열을 이용한 이진 탐색과의 비교#

장점: 데이터의 원소를 추가하거나 삭제하는 것이 용이하다.
단점: 공간 소요가 크다. (시간 복잡도가 O(log n)보다 커지는 경우가 있음)

이진 탐색 트리가 별로 효율적이지 못한 경우#

ex) 1, 2, 3, 4가 순서대로 입력된다면, 한 쪽으로 완전히 치우친 이진 탐색 트리가 되며, 여기서 4를 찾게 되는 경우 선형 탐색과 동등한 시간 복잡도 O(n)를 가지게 된다.

보다 나은 성능을 보이는 이진 탐색 트리들#

높이의 균형을 유지함으로써 O(log n)의 시간 복잡도를 보장한다.
대신 삽입과 삭제 연산이 복잡하다는 단점이 있다.

ex) AVL tree, Red-black tree

이진 탐색 트리의 추상적 자료구조#

데이터 표현#

각 노드는 (key, value)의 쌍으로 표현한다.
키를 이용하여 검색할 수 있으며 보다 복잡한 데이터 레코드로 확장할 수 있다.

연산 정의#

  1. 데이터 원소 추가: insert()
    트리에 주어진 데이터 원소를 추가한다.
  2. 특정 원소 삭제: remove()
    키를 기준으로 특정 원소를 삭제한다.
  3. 특정 원소 검색: lookup()
    키를 기준으로 특정 원소를 검색한다.
  4. 데이터 원소 나열: inorder()
    키의 순서대로 데이터 원소를 나열한다.
  5. 최소 키, 최대 키의 원소 탐색: min(), max()
    최소 키, 최대 키를 가지는 원소를 각각 탐색한다.

이진 탐색 트리의 구현#

1. 클래스 생성: Node, BinSearchTree#

class Node:
def __init__(self, key, data):
self.key = key
self.data = data
self.left = None
self.right = None
class BinSearchTree:
def __init__(self):
self.root = None

2. 데이터 원소 나열: inorder()#

깊이 우선 순회의 중위 순회와 유사하다.

class Node:
def inorder(self):
traversal = []
if self.left:
traversal += self.left.inorder()
# self를 리턴함으로써 노드들의 리스트를 리턴함
traversal.append(self)
if self.right:
traversal += self.right.inorder()
return traversal
class BinSearchTree:
def inorder(self):
if self.root:
return self.root.inorder()
else:
return []

3. 최소 키의 원소 탐색: min()#

class Node:
def min(self):
# 작은 값은 왼쪽에 있으므로 왼쪽만 탐색
if self.left:
return self.left.min()
# 왼쪽에 값이 없으면 자기 자신을 리턴
else:
return self
class BinSearchTree:
def min(self):
if self.root:
return self.root.min()
else:
return None

4. 최대 키의 원소 탐색: max()#

class Node:
def max(self):
if self.right:
return self.right.max()
else:
return self
class BinSearchTree:
def max(self):
if self.root:
return self.root.min()
else:
return None

5. 특정 원소 검색: lookup()#

찾으려는 대상의 키 값을 입력하여 노드와 부모 노드를 리턴하며, 없는 경우 각각 None을 리턴한다.

class Node:
def lookup(self, key, parent=None):
# key가 현재 노드의 key 보다 작은 경우
if key < self.key:
if self.left:
return self.left.lookup(key, self) # parent는 자기 자신
else:
return None, None
# key가 현재 노드의 key 보다 큰 경우
elif key > self.key:
if self.right:
return self.right.lookup(key, self)
else:
return None, None
# key가 현재 노드의 key와 동일한 경우
else:
return self, parent
class BinarySearchTree:
def lookup(self, key):
if self.root:
return self.root.lookup(key)
else:
return None, None

6. 데이터 원소 추가: insert()#

키와 데이터 원소를 입력받으며, 리턴 값은 없다.

class Node:
def insert(self, key, data):
if key < self.key:
if self.left:
return self.left.insert(key, data)
else:
self.left = Node(key, data)
elif key > self.key:
if self.right:
return self.right.insert(key, data)
else:
self.right = Node(key, data)
else:
raise KeyError('Duplicated Key')
class BinTreeSearch:
def insert(self, key, data):
if self.root:
return self.root.insert(key, data)
else:
self.root = Node(key, data)

7. 특정 원소 삭제: remove()#

  1. 키를 이용하여 노드를 찾는다.
    • 해당 키의 노드가 없는 경우, 삭제할 것도 없다.
    • 찾은 노드의 부모 노드도 알고 있어야 한다.
      (이 부분 때문에 lookup() 메소드 구현 시 부모 노드까지 리턴)
  2. 찾은 노드를 제거하고도 이진 탐색 트리 성질을 만족하도록 트리의 구조를 정리한다.

키를 입력받아 삭제를 수행했으면 True, 해당 키의 노드가 없으면 False를 리턴한다.

삭제되는 노드가
leaf 노드인 경우, 부모 노드의 링크를 조정하여 해당 노드를 없앤다.
자식을 하나 가지고 있는 경우, 삭제되는 노드 자리에 그 자식 노드를 대신 배치하고 부모 노드의 링크를 조정한다.
자식을 둘 가지고 있는 경우, 삭제되는 노드보다 바로 다음 키를 가지는 노드를 찾아 삭제되는 노드 자리에 대신 배치한다.
또한, 삭제되는 노드가 root node인 경우에 대해서도 처리가 필요하다.

자식의 수: countChildren()
class Node:
def countChildren(self):
count = 0
if self.left:
count += 1
if self.right:
count += 1
return count
class BinSearchTree:
def remove(self, key):
node, parent = self.lookup(key)
if node:
nChildren = node.countChildren()
# 자식이 없는 경우
if nChildren == 0:
# parent가 있으면 leaf node인 자식을 없앰
if parent:
if node == parent.left:
parent.left = None
else:
parent.right = None
# parent가 없으면 node는 root
else:
self.root = None
# 자식이 하나인 경우
elif nChildren == 1:
# 임의의 변수에 변수 할당
if node.left:
temp = node.left
else:
temp = node.right
# parent가 있으면 temp를 node 자리에 넣음
if parent:
if node == parent.left:
parent.left = temp
else:
parent.right = temp
# parent가 없으면 root에 temp 할당
else:
self.root = temp
# 자식이 둘이 경우
else:
parent = node
successor = node.right
# 삭제할 노드 바로 다음 키 값을 찾을 때까지 successor는 왼쪽 자식의 링크를 반복하여 따라감
while successor.left:
parent = successor
successor = successor.left
# node의 key와 data에 successor 대입
node.key = successor.key
node.data = successor.data
# parent와 successor의 자식 연결
if successor == parent.left:
parent.left = successor.right
else:
parent.right = successor.right
return True
else:
return False

4. 너비 우선 탐색 (BFS, Breadth-First Search)#

한 정점에서 인접한 모든 (아직 방문하지 않은) 정점을 방문하고, 방문한 각 인접 정점을 기준으로 (방문한 순서에 따라) 또다시 너비 우선 탐색을 행한다.

여기서는 큐를 이용하여 어느 정점에서 BFS를 하고 있는지를 기억하고 진행한다.

5. 깊이 우선 탐색 (DFS, Depth-First Search)#

한 정점에서 인접한 모든 (아직 방문하지 않은) 정점을 방문하되, 각 인접 정점을 기준으로 깊이 우선 탐색을 끝낸 후 다음 정점으로 진행한다.

여기서는 스택을 이용하여 어느 정점에서 DFS를 하고 있는지를 기억하고 되돌아간다.

연습문제 - 여행경로#

입출력 예#

ticketsreturn
[["ICN", "JFK"], ["HND", "IAD"], ["JFK", "HND"]]["ICN", "JFK", "HND", "IAD"]
[["ICN", "SFO"], ["ICN", "ATL"], ["SFO", "ATL"], ["ATL", "ICN"], ["ATL","SFO"]]["ICN", "ATL", "ICN", "SFO", "ATL", "SFO"]

강의 풀이#

원칙: 재귀적인 성질을 가진 한 붓 그리기 문제이며, 시작 정점은 언제나 ICN

  1. 사전을 이용하여 각 공항에서 출발하는 항공권의 리스트를 표현한다.
  2. 각 도착지를 알파벳 역순으로 정렬한다.
  3. ICN에서 갈 수 있는 공항 중 알파벳 순서로 가장 앞인 공항을 먼저 방문한다.
  4. 방문한 티켓은 제거한다.
  5. 반복한다.
def solution(tickets):
routes = {}
for t in tickets:
routes[t[0]] = routes.get(t[0], []) + [t[1]]
# O(n log n)
for r in routes:
routes[r].sort(reverse=True)
stack = ['ICN']
path = []
# O(n)
while stack:
top = stack[-1]
if top not in routes or len(routes[top]) == 0:
path.append(stack.pop())
else:
stack.append(routes[top].pop())
return path[::-1]

시간 복잡도: O(n log n)
