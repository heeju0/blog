스택 (Stacks)#

자료(data element)를 보관할 수 있는 선형 구조

후입선출(LIFO, Last-In First-Out)의 특징을 가진다.
-> 스택에서는 데이터를 넣을 때 한 쪽 끝에서 밀어 넣어야 하며(push 연산), 꺼낼 때에는 같은 쪽에서 뽑아야 한다(pop 연산).

스택의 동작#

초기 상태: 비어 있는 스택 (S = Stack())
데이터 원소 A를 스택에 추가 (S.push(A))
데이터 원소 B를 스택에 추가 (S.push(B))
데이터 원소 꺼내기 (r1 = S.pop(), r1에는 B가 담김)
데이터 원소 꺼내기 (r2 = S.pop(), r2에는 A가 담김)

스택의 오류#

스택 언더플로우 (stack underflow)#

비어 있는 스택에서 데이터 원소를 꺼내려 할 때 (r3 = S.pop())

스택 오버플로우 (stack overflow)#

꽉 찬 스택에 데이터 원소를 넣으려 할 때

스택의 추상적 자료구조 구현#

구현 방법#

방법1) 배열(array)을 이용하여 구현
-> python의 리스트와 메서드들을 이용한다.

방법2) 연결 리스트(linked list)를 이용하여 구현
-> 앞에서 구현한 양방향 연결 리스트를 이용한다.

연산 정의#

  1. 원소의 수: size()
    현재 스택에 들어 있는 데이터 원소의 수를 구한다.
  2. 스택이 비어 있는지 여부: isEmpty()
    현재 스택이 비어 있는지를 판단한다.
  3. 원소 추가: push(x)
    데이터 원소 x를 스택에 추가한다.
  4. 원소 제거: pop()
    스택의 맨 위에 저장된 데이터 원소를 리턴하고 제거한다.
  5. 원소 반환: peek()
    스택의 맨 위에 저장된 데이터 원소를 리턴한다.

배열로 스택 구현#

class ArrayStack:
# 빈 스택 초기화
def __init__(self):
self.data = []
# 스택의 크기 리턴
def size(self):
return len(self.data)
# 스택이 비어 있는지 판단
def isEmpty(self):
return self.size() == 0
# 데이터 원소 추가
def push(self, item):
self.data.append(item)
# 데이터 원소 삭제 (리턴)
def pop(self):
return self.data.pop()
# 스택의 꼭대기 원소 리턴
def peak(self):
return self.data[-1]

양방향 연결 리스트로 스택 구현#

from doublylinkedlist import Node
from doublylinkedlist import DoublyLinkedList
class LinkedListStack:
# 빈 스택 초기화
def __init__(self):
self.data = DoublyLinkedList()
# 스택의 크기 리턴
def size(self):
return self.data.getLength()
# 스택이 비어 있는지 판단
def isEmpty(self):
return self.size() == 0
# 데이터 원소 추가
def push(self, item):
node = Node(item)
self.data.insertAt(self.size()+1, node)
# 데이터 원소 삭제
def pop(self):
return self.data.popAt(self.size())
# 스택의 꼭대기 원소 리턴
def peek(self):
return self.data.getAt(self.size()).data

연습문제: 수식의 괄호 유효성 검사#

올바른 수식:
(A + B)
{(A + B) * C}
[(A + B) * (C + D)]

올바르지 않은 수식:
(A + B
A + B)
{A * (B + C})
[(A + B) * (C + D)}

알고리즘 설계

  1. 수식을 왼쪽부터 한 글자씩 읽는다.
  2. 여는 괄호 (, {, [를 만나면 스택에 푸시
  3. 닫는 괄호 ), }, ]를 만나면:
    스택이 비어 있으면 올바르지 않은 수식
    스택에서 pop, 쌍을 이루는 여는 괄호인지 검사하고, 맞지 않으면 올바르지 않은 수식
  4. 끝까지 검사한 후 스택이 비어 있어야 올바른 수식
class ArrayStack:
def __init__(self):
self.data = []
def size(self):
return len(self.data)
def isEmpty(self):
return self.size() == 0
def push(self, item):
self.data.append(item)
def pop(self):
return self.data.pop()
def peek(self):
return self.data[-1]
def solution(expr):
match = {
')': '(',
'}': '{',
']': '['
}
S = ArrayStack()
for c in expr:
if c in '({[':
S.push(c)
elif c in match:
if S.isEmpty():
return False
else:
t = S.pop()
if t != match[c]:
return False
return S.isEmpty()

스택의 응용: 수식의 후위 표기법#

중위 표기법과 후위 표기법#

중위 표기법 (infix notation)#

연산자가 피연산자들의 사이에 위치하는 표기법
ex) (A + B) * (C + D)
ㅤㅤ ㅤ①ㅤ ③ ㅤ②

후위 표기법 (postfix notation)#

연산자가 피연산자들의 뒤에 위치하는 표기법
ex) A B + C D + *
ㅤㅤ ㅤ ① ㅤ ② ③

중위 표현식 -> 후위 표현식#

변환 방법#

중위 표현식의 맨 왼쪽부터 확인한다.

  1. 피연산자의 경우 그대로 옮겨 적고,
  2. 연산자의 경우
    • 스택이 비어 있으면 스택에 연산자를 push
    • 비어 있지 않으면
      • 스택에 있는 연산자의 우선순위가 새로운 연산자보다 높거나 같으면 pop
      • 새로운 연산자의 우선순위가 더 높으면 push
예제
[중위] A * B + C -> [후위] A B * C +
[중위] A + B * C -> [후위] A B C * +
[중위] A + B + C -> [후위] A B + C +

괄호의 처리#

괄호가 있는 경우, 여는 괄호는 스택에 push하고, 닫는 괄호는 다시 여는 괄호가 등장할 때까지 pop한다. 이때, 연산자를 만날 경우 여는 괄호 이후까지 pop하지 않도록 여는 괄호의 우선순위를 가장 낮게 설정한다.

예제
[중위] (A + B) * C -> [후위] A B + C *
[중위] A * (B + C) -> [후위] A B C + *
[중위] (A + B) * (C + D) -> [후위] A B + C D + *
[중위] (A + (B - C)) * D -> [후위] A B C - + D *
[중위] A * (B - (C + D)) -> [후위] A B C D + - *

알고리즘 설계#

1. 연산자의 우선순위 설정#

prec = {
'*': 3, '/': 3,
'+': 2, '-': 2,
'(': 1
}

2. 중위 표현#

중위 표현식을 왼쪽부터 한 글자씩 읽어서,

  1. 피연산자이면 그냥 출력
    '('이면 스택에 push
    ')'이면 '('이 나올 때까지 스택에서 pop하고 출력
  2. 연산자이면 스택에서 우선순위가 더 높거나 같은 값들을 pop, 출력
    그리고 이 연산자는 스택에 push

Hint)
스택의 맨 위에 있는 연산자와의 우선순위 비교: peek() 연산 이용
스택에 남아 있는 연산자 모두 pop()하는 순환문: while not opStack.isEmpty()

❗ 마지막 피연산자 이후의 연산자들이 answer에 입력이 안 돼서 고생했다.
for문 이후에 while not opStack.isEmpty()를 추가했더니 해결됐지만 코드가 깔끔하진 않은 느낌..

def solution(S):
answer = ''
opStack = ArrayStack()
for i in S:
# '('의 경우 스택에 push
if i == '(':
opStack.push(i)
# ')'의 경우 '('이 나올 때까지 스택에서 pop하여 answer에 추가
elif i == ')':
while opStack.peek() != '(':
answer += opStack.pop()
# '(' pop
opStack.pop()
# 피연산자의 경우 바로 answer에 추가
elif i not in prec:
answer += i
# 연산자의 경우
else:
# 스택이 비어 있지 않다면
if not opStack.isEmpty():
t = opStack.peek()
# 스택의 맨 위의 값이 현재 값 i보다 크거나 같을 때,
if prec[t] >= prec[i]:
# 스택의 모든 연산자를 pop하여 answer에 추가
while not opStack.isEmpty():
answer += opStack.pop()
# 모든 연산자에 대해 스택에 push
opStack.push(i)
while not opStack.isEmpty():
answer += opStack.pop()
return answer

스택의 응용: 후위 표기 수식 계산#

후위 표현식 계산#

후위 표현식의 맨 왼쪽부터 확인한다.

  1. 피연산자의 경우 스택에 push
  2. 연산자의 경우 스택의 모든 값을 pop하여 연산한 뒤 다시 스택에 push
    • 연산 시, 덧셈(+), 곱셈(*)은 순서가 중요하지 않지만,
    • 뺄셈(-), 나눗셈(/)은 반드시 먼저 스택에 push된 값부터 적용해야 한다.
  3. 연산이 마무리되면, 스택에는 하나의 원소만 남게 되고 그 값이 결과 값이 된다.

ex) A B + C D + * -> (A + B) * (C + D)

알고리즘 설계#

후위 표현식을 왼쪽부터 한 글자씩 읽어서
피연산자면 스택에 push,
연산자면 스택에서 pop -> (1), pop -> (2) 후 (2) 연산 (1)을 계산한 뒤, 이 결과를 다시 스택에 push
수식의 끝에 도달하면 스택에서 pop하여 계산 결과로 사용

숫자와 연산자를 나누는 함수 구현
# exprStr은 중위 표현식으로 표현된 수식
def splitTokens(exprStr):
tokens = []
var = 0
varProcessing = False # 숫자인지에 대한 flag 역할
for c in exprStr:
# 공백이 있으면 넘어감
if c == ' ':
continue
# 숫자인 경우 기존의 var 값에 10을 곱해 1의 자리에 c 입력
if c in '0123456789':
val = val * 10 + int(c)
valProcessing = True
# 연산자인 경우
else:
# 이전에 숫자가 있었던 경우 tokens에 값을 넘기고 초기화
if valProcessing:
tokens.append(val)
val = 0
valProcessing = False
# tokens에 연산자 추가
tokens.append(c)
# tokens에 넣지 않은 숫자 넣어주기
if valProcessing:
tokens.append(val)
return tokens
