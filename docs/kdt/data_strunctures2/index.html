1. 연결 리스트 (Linked Lists)#

선형 배열과 유사한 구조

1-1. 배열 vs. 연결 리스트#

ㅤ배열리스트
저장 공간연속한 위치임의의 위치
특정 원소 지칭매우 간편 (인덱스)선형탐색과 유사 (k번째 값을 찾음)
시간 복잡도$O(1)$$O(n)$

1-2. 추상적 자료구조 (Abstract Data Structures)#

자료구조의 내부 구현은 숨기고, 외부에서 보이는 것(데이터, 연산)을 제공하는 구조

Data: 정수, 문자열, 레코드 등
A set of operations: 삽입, 삭제, 순회, 정렬, 탐색 등

1-3. 기본적 연결 리스트#


앞의 화살표가 뒤의 값을 가리키도록 구성된 리스트 형태
각 아이템을 노드라고 하며, Node 하나에는 Data 다음 노드를 가리키는 Link 가지고 있다.

(Node 내의 데이터는 문자열, 레코드, 또 다른 연결 리스트 등 다른 구조로 이루어질 수 있으나, 여기서는 간단하게 숫자 데이터에 대해서만 다룬다.)

연결 리스트에서 알아야 할 요소들#

Head: 리스트의 맨 앞
Tail: 리스트의 맨 끝
# of nodes: 노드의 개수

이 요소들과 리스트가 가질 수 있는 연산들을 저장한 것이 이 리스트의 추상적 자료구조가 된다.

연결 리스트의 추상적 자료구조 생성#

1. 두 개의 클래스 생성#

Node 클래스: 데이터(data), 링크(next)

class Node:
def __init__(self, item):
self.data = item
self.next = None # 아직 아무 값도 가리키고 있지 않음

LinkedList 클래스: 노드의 개수(nodeCount), 맨 앞(head), 맨 끝(tail)

class LinkedList:
def __init__(self):
self.nodeCount = 0
self.head = None # 아직 아무 값도 가리키고 있지 않음
self.tail = None # 아직 아무 값도 가리키고 있지 않음

2. 추상적 자료구조에 추가할 수 있는 연산 정의#

1) 특정 원소 참조 (k번째)
2) 리스트 순회: 차례로 노드들을 방문하는 연산
3) 길이 얻어내기
4) 원소 삽입
5) 원소 삭제
6) 두 리스트 연결

3. 특정 원소 참조#

원소의 순서는 1부터 시작하도록 지정한다. (0번은 다른 목적으로 사용하고자 한다.)

def getAt(self, pos):
if pos < 1 or pos > self.nodeCount:
return None
i = 1
curr = self.head
while i < pos:
curr = curr.next
i += 1
return curr

4. 리스트 순회#

def traverse(self):
lst = []
curr = self.head
while curr:
lst.append(curr.data)
curr = curr.next
return lst

5. 길이 얻어내기#

nodeCount를 리턴하는 단순한 과정

6. 원소 삽입#

pos가 가리키는 위치에 newNode를 삽입하고, 성공이면 True, 실패이면 False 리턴
삽입하고자 하는 위치가 리스트의 맨 앞인 경우에는, prev가 없으며 head를 조정해야 할 필요가 있다.
삽입하고자 하는 위치가 리스트의 맨 끝인 경우에는 tail을 조정해야 할 필요가 있다. 또한, 이 경우 맨 앞에서부터 찾아갈 필요가 없이 바로 tail을 찾을 수 있다.

def insertAt(self, pos, newNode):
if pos < 1 or pos > self.nodeCound + 1:
return False
# 리스트의 맨 앞인 경우
if pos == 1:
newNode.next = self.head
self.head = newNode
else:
# 리스트의 맨 끝인 경우 getAt()을 사용하지 않고 바로 tail을 적용
if pos == self.nodeCount + 1:
prev = self.tail
# 중간인 경우
else:
prev = self.getAt(pos-1)
newNode.next = prev.next
prev.next = newNode
# 리스트의 맨 끝인 경우
if pos == self.nodeCount + 1:
self.tail = newNode
self.nodeCount += 1
return True

원소 삽입의 시간 복잡도는
맨 앞에 삽입하는 경우 $O(1)$
중간에 삽입하는 경우 $O(n)$ (평균적으로 리스트의 길이가 길어질수록 복잡도가 커짐)
맨 끝에 삽입하는 경우 $O(1)$

7. 원소 삭제#

pos가 가리키는 위치의 node를 삭제하고, 그 node의 데이터를 리턴
삭제하고자 하는 노드가 맨 앞에 있을 때는 prev가 없으며 head를 조정해야 한다.
삭제하고자 하는 노드가 맨 뒤에 있을 때는 tail을 조정해야 한다.

def popAt(self, pos):
if pos < 1 or pos > self.nodeCount:
raise IndexError
curr = self.getAt(pos)
# 노드가 하나인 경우
if self.nodeCount == 1:
self.head = None
self.tail = None
# 리스트의 맨 앞인 경우
elif pos == 1:
self.head = curr.next
else:
prev = self.getAt(pos-1)
# 리스트의 맨 뒤인 경우
if pos == self.nodeCount:
prev.next = None
self.tail = prev
# 중간인 경우
else:
prev.next = curr.next
self.nodeCount -= 1
return curr.data

원소 삭제의 시간 복잡도는
맨 앞에서 삭제하는 경우 $O(1)$
중간에서 삭제하는 경우 $O(n)$
맨 뒤에서 삭제하는 경우 $O(n)$

8. 두 리스트 연결#

def concat(self, L):
self.tail.next = L.head
# L이 []인 경우를 고려해야 함
if L.tail:
self.tail = L.tail
self.nodeCount += L.nodeCount

연결 리스트 최종#

self.next = None
class LinkedList:
def __init__(self):
self.nodeCount = 0
self.head = None
self.tail = None
def __repr__(self):
if self.nodeCount == 0:
return 'LinkedList: empty'
s = ''
curr = self.head
while curr is not None:
s += repr(curr.data)
if curr.next is not None:
s += ' -> '
curr = curr.next
return s
# 특정 원소 참조
def getAt(self, pos):
if pos < 1 or pos > self.nodeCount:
return None
i = 1
curr = self.head
while i < pos:
curr = curr.next
i += 1
return curr

def traverse(self):
result = []
curr = self.head
while curr:
result.append(curr.data)
curr = curr.next

def getLength(self):
return self.nodeCount

self.head = newNode
