django를 활용하여 웹 사이트 만들기

💡 『KDT 프로그래머스 인공지능 데브코스』에서 공부중..👀

우리의 목표

django를 활용하여 웹 사이트를 구축한다.

1. django#

django는 파이썬 기반 웹 프레임워크 중 하나이며, pinterest, instagram 등이 django를 사용한다.

django vs. Flask#

둘다 파이썬 기반의 웹 프레임워크지만 지향성이 조금 다르다.
Flask는 "마이크로" 웹 프레임워크로 최소한의 기능을 가지고 있으며 추가적인 모듈을 추가하면서 빌드업을 한다.
반면, django는 거의 모든 것이 이미 내장되어 있어 프로젝트 사이즈가 Flask보다 크며, 그 안에 우리가 원하는 부분들을 채워 나가는 방식이다.

가상환경 활용#

가상환경 설치
$ pip install virtualenv
가상환경 venv 생성
$ virtualenv venv
가상환경 venv 활성화
$ venv\Scripts\activate.bat

django 설치하기#

django 모듈 설치
$ pip install django

django의 구성 요소#

Project는 여러 개의 App으로 구성되며, App은 특정 명령을 수행하는 View나 Template의 목록이다.
ex) 네이버에는 sports, blog, cafe등이 있고, django는 이들을 sports App, blog App, cafe App 등 App 단위로 관리를 진행한다.

모듈화 진행을 통해 각 기능을 독립적으로 개발할 수 있다.

django 시작하기#

Project 만들기#

프로젝트 webproj 시작
$ django-admin startproject webproj
$ cd webproj
$ dir

manage.py 파일을 통해 서버를 가동한다.

서버 가동
python manage.py runserver

VS Code로 우리가 생성한 webproj 폴더를 열어서 구성 요소를 살펴본다.

webproj 하위의 webproj 폴더#

__init__.py: webproj라는 디렉토리가 파이썬 모듈로 인식되게 하는 역할

asgi.py, wsgi.py: 서버에서 해당 django 프로젝트를 가동할 때 다루는 부분

settings.py: 파이썬 django 프로젝트에 설정 사항을 반영하는 파일

settings.py의 요소설명
SECRET_KEY암호 키
DEBUG디버깅 모드 실행 유무 (True/False)
ALLOWED_HOSTSdjango 프로젝트에 접근하도록 허용된 주소 목록
INSTALLED_APPSdjango 프로젝트의 여러 가지 앱들 중 설치하고자 하는 앱 목록
MIDDLEWAREdjango 프로젝트의 여러 가지 미들웨어 중 설치하고자 하는 미들웨어 목록
ROOT_URLCONF프로젝트에서 url 모듈을 관리할 모듈 설정
TEMPLATES실제 보는 화면에 대한 요소들에 대한 설정
WSGI_APPLICATION파이썬 상에서 웹 서버와 소통하는 사이에 필요한 애플리케이션 설정
DATABASES프로젝트 상에서 다루게 되는 여러 가지 자료들을 저장하는 곳을 설정 (기본값: db.sqlite3)
AUTH_PASSWORD_VALIDATORS패스워드를 관리하는 부분
STATIC_URLCSS, JavaScript, Image 등의 정적 파일들을 담아둘 폴더 설정

Flask보다 많은 기능이 이미 내장되어 있다는 것을 확인할 수 있다. 이 기능들을 적절히 활용하여 매우 빠르게 웹 애플리케이션과 웹 사이트를 구축할 수 있다.

urls.py: url을 관리하는 부분

django App 만들기#

앱을 만들 때에는 프로젝트 폴더 안에서 진행한다.

홈페이지를 보여주는 hompage App 생성
$ django-admin startapp homepage

하위에 webproj뿐만 아니라 homepage라는 디렉토리가 추가된다.

webproj 하위의 homepage 폴더#

admin.py: 파이썬이 기본적으로 제공하는 admin 페이지에 관한 부분
apps.py: 앱에 대한 설정을 관리하는 부분
models.py: homepage 모듈 내에서 사용될 DB의 스키마 등을 클래스 형태로 작성할 수 있는 부분
tests.py: 이 프로젝트의 테스트 케이스에 대해 설명
views.py: homepage 앱에서 view를 관리하는 방식에 관한 부분

django의 MVT 패턴#

디자인 패턴: 코드의 모듈화를 통해 각 코드가 독립적으로 동작하여 유기적으로 연결되도록 해주는 구조

django에서는 MVC(Model-View-Controller) 패턴을 바탕으로 MVT 패턴을 만들어 사용한다.

MVT 패턴#

한 유저가 요청을 보내면 서버(Django)는 URL을 인식하여 어떤 경로에 대한 요청인지 확인하고(urls.py), 존재하는 경로인 경우 View에 보내 요청을 처리한다(views.py).
데이터베이스를 처리하는 경우에는 Model이 DB를 관리하며 DB와 소통한다. Django는 DB를 ORM 구조로 관리한다.
웹 문서를 보여주는 경우에는 Template이 관리한다. Template 언어를 이용하면 View에서 처리한 여러 변수 등을 html에 내장하여 함께 출력한다. 일반적으로 html 내에서 로직이나 변수를 사용할 수 없는데 Template 언어를 사용하여 이 부분을 해결할 수 있다.
View - Model 혹은 View - Template 간의 상호작용이 끝나면 이 결과를 다시 Django를 거쳐 유저에게 보여진다.

3. Template로 보여줄 화면 구성하기#

HTML 파일 연동#

views.py
def index(request):
return HttpResponse('<h1>Hello World!</h1>')

위와 같이 HttpResponse 메서드 내부에 마크업 언어를 적용할 수 있으나, 문장이 길어지면 관리가 복잡해진다. django에서는 html 문서를 별도로 관리할 수 있는 함수 render를 제공하며 이 함수는 세 개의 파라미터를 인자로 받는다.

view.py
def index(request):
return HttpResponse('<h1>Hello World!</h1>')
render(request, 'index.html', {})
render 메서드의 파라미터

request = 함수에 입력된 인자
'index.html' = 응답하는 과정에서 보여줄 파일 지정
{} = request와 파일을 처리하는 과정에서 사용할 여러 인자들의 딕셔너리 형태

Template은 Model과 View와는 다르게 파이썬 파일이 아닌 html 파일을 만들며, 이 html 파일을 관리하는 방법은 여러 가지가 있으나 여기서는 각 앱의 디렉토리에서 관리하는 방법을 배운다.

Template 문법 활용#

위와 같이 template 언어를 사용할 때는 중괄호 두 개를 사용한다. import와 유사한 구조로, my_num이 존재하면 그 값을 보여주는 문법이다.

Template변수|length, Template변수|upper 등 적용 가능

index.html
<body>

<p>blah blah blah</p>

</body>

Template 태그 ({% tag %} {% endtag %})#

html 파일 자체에는 프로그래밍적 로직을 도입할 수 없다. 그러나 django를 사용하는 과정에서는 django template 언어를 사용할 수 있고, 여기에는 template 태그라는 기능이 있다.

views.py
def index(request):
name = 'Michael'
nums = [1, 2, 3, 4, 5]
return render(request, 'index.html', {'my_name':name, 'my_list':nums})
index.html
<body>
<h1>Title</h1>
<p>blah blah blah</p> <!-- 문단 태그 -->
<p>{{ my_name|upper }}</p>
{% for element in my_list %}
{% if not element|divisibleby:"2" %}
<p>{{ element }}</p>
{% endif %}
{% endfor %}
</body>

nums 리스트에 담긴 원소들 중 홀수인 원소만 나타냈다.

Template에서 모델 정보 갱신하기#

Form을 만들어주어야 하는데 ModelForm은 어떤 모델에 대해 입력 칸을 만들어 주는 객체이다.

homepage 폴더에 forms.py 파일 생성

forms.py
from django import forms
from .models import Coffee # Model 호출
# Modelform을 상속받는 CoffeeForm 생성
class CoffeeForm(forms.ModelForm):
class Meta:
# Form을 만들기 위해 어떤 모델을 사용해야 하는지 지정
model = Coffee
fields = ('name', 'price', 'is_ice')

만든 정보를 View에서 Template으로 전달한다.

views.py
from .models import Coffee
from .forms import CoffeeForm
def coffee_view(request):
coffee_all = Coffee.objects.all()
form = CoffeeForm()
return render(request, 'coffee.html', {"coffee_list": coffee_all, "coffee_form": form})

Form을 만들고, 입력하는 버튼을 추가한다. Form에서 담은 내용을 전달하기 위해 메서드는 POST로 지정하고, Form에 대한 보안 처리가 필요하기 때문에 서버의 입장에서 CSRF 토큰을 Form 안에 삽입한다.

coffee.html
<!DOCTYPE html>
<html>
<head>
<title>Coffee List</title>
</head>
<body>
<h1>My Coffee List</h1>
{% for coffee in coffee_list %}
<p>{{ coffee.name }}, {{ coffee.price }}</p>
{% endfor %}
<form method="POST"> {% csrf_token %}
{{ coffee_form.as_p }}

def coffee_view(request):

form = CoffeeForm()
